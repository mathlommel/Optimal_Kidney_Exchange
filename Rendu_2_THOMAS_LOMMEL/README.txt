- RENDU N°1 : PARTIE DETERMINISTE

Dans le fichier .zip fournit, vous trouverez le code demandé, qui est segmenté en plusieurs parties.

Le code principal est placé dans le notebook intitulé "Projet_OSI_THOMAS_LOMMEL". Dans celui-ci, vous
trouverez le squelette du notebook initial du projet, complété par les codes et les explications nécessaires.

Pour toutes les parties du projet, nous avons décidé de segmenter le rendu. Ainsi, vous trouverez plusieurs 
dossiers ("Formulations_compactes", "Generation_de_colonnes" et "Tests"), dans lesquels un ou plusieurs 
notebooks sont présents.
Les codes et fonctions construites dans ces notebooks sont importées dans le notebook principal, au moment
opportun.

Les instances de tests sont elles placées dans le dossier "Data_KEP".

Aussi, pour faciliter l'explication, certaines images sont présentes, et affichées dans le notebook. 
Celles-ci sont présentes dans le dossier "Images".

Au sein de ce dossier se trouvent également des captures d'écran des graphes de performances. Ces images sont
affichées dans le notebook, présentes au cas où le graphique associé aurait disparu, et pour éviter de devoir
relancer le code.


- RENDU N°2 : PARTIE INCERTITUDES

Dans le fichier .zip fournit, vous trouverez le code demandé, qui est toujours segmenté en plusieurs parties.
Toutes les nouvelles implémentations/fonctions/tests sont situés dans le dossier "Formulations_avec_incertitudes".
Comme dit précédemment, les codes et fonctions construites dans ces notebooks seront importées dans le notebook 
principal, au moment opportun.

Le code principal de cette seconde partie est placé dans le notebook intitulé "Projet_OSI_THOMAS_LOMMEL", à la 
suite des codes de la première partie. 

Toujours pour faciliter l'explication, certaines images sont affichées dans le notebook, et sont stockées dans le
même dossier que pour le rendu n°1.



Toutes les sorties du notebook principal ont été gardées, et devraient normalement être visibles. Dans le 
cas où ce ne serait pas le cas, une version .pdf est aussi présente dans le fichier .zip
Attention toutefois, dans la version pdf, les images importées ne sont pas présentes, et certaines formules
